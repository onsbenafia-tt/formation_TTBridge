Introduction
Le portail des amicales est une interface dédiée permettant aux amicales de gérer efficacement leurs conventions et leurs adhérents au sein de la plateforme TTbridge. À travers cet espace, les utilisateurs peuvent consulter leurs conventions, suivre et administrer les informations relatives à leurs adhérents, ainsi que gérer les demandes d’offres soumises. Le portail offre également des fonctionnalités d’export de données, facilitant ainsi le suivi, l’analyse et le reporting des demandes.

Authentification

Authentification

L’accès au portail TTBridge – Amicale s’effectue via une interface d’authentification sécurisée.

L’utilisateur doit renseigner ses identifiants de connexion dans un formulaire dédié comportant les champs suivants :

  • Nom d’utilisateur (obligatoire) : identifiant fourni par l’administrateur ou adresse email.
  • Mot de passe (obligatoire) : mot de passe associé au compte utilisateur.

Une icône intégrée au champ du mot de passe permet d’afficher ou de masquer la saisie afin de vérifier son contenu.

En cas d’oubli, le lien « Mot de passe oublié ? » permet de lancer la procédure de réinitialisation.

Après saisie des informations, l’utilisateur clique sur le bouton « Se connecter » pour accéder à l’application.

L’accès est autorisé uniquement si les identifiants saisis sont valides.

L’accès n’est autorisé que si les identifiants sont valides.

Tableau de bord

Le tableau de bord constitue la page d’accueil principale après authentification et offre une vue globale des informations de l’amicale.

L’utilisateur peut basculer entre le mode clair (Light) et sombre (Dark) via le bouton situé en haut à droite.

  • Nombre total d’adhérents

  • Total des demandes
  • Demandes activées
  • Demandes en attente d’activation
  • Demandes en attente de confirmation
  • Demandes annulées

1. Consultation des conventions

Liste des conventions

Cette interface permet de visualiser les conventions enregistrées dans le système.

Chaque convention affiche son nom, le responsable de paiement, la date de fin et le statut.

Le statut permet d’identifier rapidement si la convention est active ou expirée.

Une action permet de consulter les détails de chaque convention.

La pagination en bas de page permet de naviguer entre les différentes conventions.

Filtrer les conventions

On peut filtrer la liste des conventions en saisissant le nom ou en sélectionnant une plage de dates.

Des filtres supplémentaires permettent de trier par statut ou responsable de paiement.

Cliquer sur Appliquer pour afficher les résultats correspondants.

1. Gestion des adhérents

Liste des adhérents

Cette interface permet de consulter les adhérents enregistrés dans le système.

On peut rechercher un adhérent en saisissant un mot clé (CIN, CNRPS ou numéro de téléphone).

Le tableau affiche les informations principales telles que le CIN, le nom, le prénom, le numéro de téléphone, la date de naissance, le CNRPS et la région.

Des actions permettent de consulter, modifier ou supprimer un adhérent.

La pagination en bas de page permet de naviguer entre les différents résultats.

2. Ajout des adhérents

L’ajout des adhérents peut se faire de deux manières : manuellement ou par importation Excel.

Ajout manuel : l’utilisateur peut renseigner les informations de l’adhérent via un formulaire dédié (CIN, nom, prénom, date de naissance, téléphone, CNRPS et région).

Certains champs sont facultatifs comme le matricule ou le rattachement.

Cliquer sur le bouton Ajouter pour enregistrer l’adhérent ou Annuler pour abandonner.

Importation Excel : il est possible d’importer plusieurs adhérents en une seule opération en sélectionnant un fichier Excel.

Le fichier Excel doit respecter un format prédéfini contenant les colonnes suivantes : CIN, matricule, rattachement, prénom, nom, téléphone, date de naissance, CNRPS et région .

Cliquer sur Choisir un fichier Excel, puis sur Importer pour lancer l’opération.

Cette fonctionnalité permet de gérer rapidement un grand nombre d’adhérents.

Détails de l’adhérent

1. Consultation des demandes

Liste des demandes

Cette interface permet de consulter l’ensemble des demandes associées aux adhérents.

On peut rechercher une demande en saisissant un mot clé tel que l’identifiant de transaction ou les informations de l’adhérent.

Le tableau affiche les principales informations telles que l’identifiant de la demande, l’adhérent, le produit et le statut.

Les statuts permettent de suivre l’évolution des demandes : activée, en attente d’activation, en attente de confirmation, ou annulée.

Une action permet de consulter les détails de chaque demande.

Il est également possible d’exporter les données pour un traitement externe.

La pagination permet de naviguer entre les différentes pages de résultats.

2. Exportation des demandes

Exporter les transactions

Cette fonctionnalité permet d’exporter les demandes selon des critères spécifiques.

L’utilisateur peut définir une période en sélectionnant une date de début et une date de fin.

Un filtre par statut permet de choisir les demandes à exporter : activées, en attente de confirmation, en attente d’activation ou annulées.

Il est également possible de sélectionner l’option Tous pour exporter l’ensemble des demandes.

Cliquer sur le bouton Exporter pour générer le fichier contenant les données filtrées.

Cette fonctionnalité facilite l’analyse et le traitement des transactions en dehors du système.

3. Détails des demandes

Détails de la demande

Cette interface permet de consulter les informations détaillées d’une demande sélectionnée.

La section Détails de la demande affiche l’identifiant, le statut et la source de la demande.

La section Détails de l’adhérent présente les informations associées telles que le nom, le CIN, le CNRPS, le téléphone, la date de naissance et la région.

La section Détails du produit affiche les caractéristiques du produit souscrit, incluant le nom, la subvention, les montants et la durée d’engagement.

Une zone Suivi de la transaction permet de visualiser l’avancement de la demande (création, activation, livraison du voucher).

Un bouton Corriger les données de l’adhérent permet de modifier les informations en cas d’erreur.

Valider ou rejeter une demande

Lorsqu’une demande est au statut « En attente de confirmation amicale », l’utilisateur doit la traiter en la validant ou en la rejetant.

Pour accéder au traitement, cliquer sur l’icône de consultation de la demande.

Une fenêtre s’ouvre affichant les informations détaillées :

  • Détails de la demande : identifiant, statut et source
  • Détails de l’adhérent : informations personnelles
  • Détails du produit : caractéristiques de l’offre

Avant de prendre une décision, il est possible de :

  • Corriger les données de l’adhérent : modifier les informations si nécessaire
  • Télécharger : consulter les documents associés à la demande

Deux actions principales sont disponibles :

  • Approuver : valider la demande pour passer à l’étape suivante (activation)
  • Rejeter : refuser la demande

Après validation, deux cas sont possibles :

  • Demande approuvée : la demande passe au statut « En attente d’activation »
  • Demande rejetée : la demande passe au statut « Annulée » et est marquée comme refusée dans le suivi de la transaction

Le suivi de la transaction est automatiquement mis à jour pour refléter l’état final de la demande.

Cliquer sur l’icône de fermeture pour quitter la fenêtre.

Création d’une demande

Cette interface permet de créer une nouvelle demande en sélectionnant une convention et un produit.

L’utilisateur peut rechercher un adhérent en saisissant un mot clé (CIN, CNRPS, matricule ou nom).

Un champ facultatif permet de saisir le numéro d’appel lié à l’offre.

Il est possible d’ajouter un adhérent directement depuis cette interface en cliquant sur le bouton Ajouter adhérent.

L’ajout peut se faire :
- manuellement en remplissant le formulaire (CIN, nom, prénom, téléphone, CNRPS, région, etc.)
- ou par importation d’un fichier Excel pour ajouter plusieurs adhérents en une seule opération

Cliquer sur Importer après sélection du fichier Excel pour lancer l’importation.

Une fois les informations complètes, cliquer sur Demander pour valider la création de la demande.

Importer des adhérents depuis Excel

Cette fonctionnalité permet d’importer plusieurs adhérents en une seule opération à partir d’un fichier Excel.

L’utilisateur clique sur Choisir un fichier Excel pour sélectionner le fichier à importer.

Le fichier Excel doit respecter un format prédéfini contenant les colonnes suivantes : CIN, matricule, rattachement, prénom, nom, téléphone, date de naissance, CNRPS, région et MSISDN.

Chaque ligne du fichier correspond à un adhérent à ajouter dans le système.

Cliquer sur Importer pour lancer le traitement du fichier ou Annuler pour interrompre l’opération.

Cette méthode permet d’importer rapidement un grand volume d’adhérents tout en garantissant la cohérence des données.

Changer le mot de passe

Pour modifier le mot de passe :

  • Saisir le nouveau mot de passe
  • Confirmer le mot de passe dans le champ de confirmation

Une icône permet d’afficher ou de masquer le mot de passe pour vérifier la saisie.

Cliquer sur « Mettre à jour » pour enregistrer le nouveau mot de passe.

L’utilisateur peut également accéder à la fonctionnalité via le menu « Changer mot de passe » dans l’application.

En cas d’erreur ou si le mot de passe ne respecte pas les règles, un message d’alerte s’affiche.

Réinitialiser le mot de passe

En cas d’oubli du mot de passe, cliquer sur le lien « Mot de passe oublié ? » depuis la page de connexion.

Une page de réinitialisation s’affiche. L’utilisateur doit saisir son nom d’utilisateur ou son email.

Cliquer sur le bouton « Envoyer le lien » pour recevoir un lien de réinitialisation.

Après réception du lien et accès à la page dédiée, l’utilisateur est invité à définir un nouveau mot de passe.

Ajouter un utilisateur

Pour créer un nouvel utilisateur, accéder au menu « Ajouter utilisateur » depuis le tableau de bord.

Une fenêtre de saisie s’affiche permettant de renseigner les informations du nouvel utilisateur.

Remplir les champs obligatoires suivants :

  • Prénom : saisir le prénom de l’utilisateur
  • Nom : saisir le nom de l’utilisateur
  • Nom d’utilisateur : identifiant unique de connexion
  • Email : adresse email de l’utilisateur

Sélectionner ensuite le rôle de l’utilisateur via la liste déroulante :

  • super_admins : accès complet à toutes les fonctionnalités
  • admins : gestion opérationnelle avec droits étendus
  • viewers : accès en consultation uniquement

Après avoir renseigné toutes les informations, cliquer sur le bouton Ajouter pour enregistrer l’utilisateur.
